## System Architecture

The Number Base Converter uses a simple but effective architecture with three main components:

1. **PIC18F4550 Microcontroller** - Central processing unit
2. **HDM64GS12 Graphic LCD** - Display output
3. **4×4 Matrix Keypad** - User input

```
┌─────────────────┐
│  PIC18F4550     │
│                 │
│  PORTB ─────────┼─────► 4×4 Keypad
│                 │        (Matrix Interface)
│  PORTD ─────────┼─────► HDM64GS12 LCD
│                 │        (Data Bus)
│  PORTA/C/E ─────┼─────► LCD Control
│                 │        (Control Lines)
└─────────────────┘
```

## Keypad Pin Mappings

### PORTB Configuration

The 4×4 matrix keypad is connected exclusively to **PORTB** of the PIC18F4550.

```c
#define use_portb_kbd TRUE
#byte kbd = 0xF81  // PORTB address for PCH compiler
```

### Black Keypad Pin Definitions

The default configuration uses the black keypad layout:

```c
// Column pins (outputs)
#define COL0 (1 << 4)  // Bit 4 = RB4
#define COL1 (1 << 5)  // Bit 5 = RB5
#define COL2 (1 << 6)  // Bit 6 = RB6
#define COL3 (1 << 7)  // Bit 7 = RB7

// Row pins (inputs)
#define ROW0 (1 << 0)  // Bit 0 = RB0
#define ROW1 (1 << 1)  // Bit 1 = RB1
#define ROW2 (1 << 2)  // Bit 2 = RB2
#define ROW3 (1 << 3)  // Bit 3 = RB3

// Composite definitions
#define ALL_ROWS (ROW0|ROW1|ROW2|ROW3)
#define ALL_PINS (ALL_ROWS|COL0|COL1|COL2|COL3)
```

### Alternative Blue Keypad Configuration

An alternative pin mapping is available for blue keypad variants:

```c
#ifdef blue_keypad
#define COL0 (1 << 2)  // RB2
#define COL1 (1 << 3)  // RB3
#define COL2 (1 << 6)  // RB6
// Note: COL3 not defined for 3-column keypad

#define ROW0 (1 << 4)  // RB4
#define ROW1 (1 << 7)  // RB7
#define ROW2 (1 << 1)  // RB1
#define ROW3 (1 << 5)  // RB5
#endif
```

<Note>
  To use the blue keypad configuration, add `#define blue_keypad` before including the kbd4x4.c library.
</Note>

## Matrix Keypad Scanning

### Scanning Algorithm

The keypad uses a column-scanning technique:

1. **Column Selection**: Set one column LOW, others HIGH (via TRIS direction control)
2. **Row Reading**: Read all row inputs
3. **Key Detection**: If any row is LOW, a key in that column is pressed
4. **Key Identification**: Determine which row is LOW to identify the specific key

### Port Direction Control

```c
#define set_tris_kbd(x) set_tris_b(x)

// Example: Scan column 0
set_tris_kbd(ALL_PINS&~COL0);  // Set COL0 as output, others as input
kbd=~COL0&ALL_PINS;             // Drive COL0 LOW, others HIGH

// Read the port
if((kbd & (ALL_ROWS))!=(ALL_ROWS)) {
    // A key in column 0 is pressed
    // Check which row...
}
```

## LCD Connections

### HDM64GS12 Interface

The graphic LCD uses a parallel interface with data and control lines.

<Note>
  Exact pin mappings for the LCD are defined in the `hdm64gs12.c` library file. The library handles low-level communication with the KS0108/KS0107 controllers.
</Note>

### Typical LCD Pin Usage

Based on standard KS0108 LCD connections:

| LCD Pin | Function | Typical PIC Connection |
|---------|----------|------------------------|
| DB0-DB7 | Data Bus | PORTD (parallel 8-bit) |
| RS      | Register Select | PORTA or PORTC pin |
| R/W     | Read/Write | PORTA or PORTC pin |
| E       | Enable | PORTA or PORTC pin |
| CS1     | Chip Select 1 | PORTA or PORTC pin |
| CS2     | Chip Select 2 | PORTA or PORTC pin |
| RST     | Reset | PORTA or PORTC pin |

### LCD Library Usage

```c
#include <hdm64gs12.c>  // Low-level LCD driver
#include <graphics.c>    // High-level graphics functions

// Initialization
glcd_init(on);
glcd_fillscreen(0);  // Clear screen

// Text display (5×7 font)
glcd_text57(x, y, text, size, mode);
```

```
        │
        ├────► LCD VSS
        │
        └────► Common ground
```

### Decoupling Capacitors

<Note>
  Add 0.1µF ceramic capacitors between VDD and VSS pins on the PIC18F4550, placed as close to the IC as possible. Consider a 10µF electrolytic capacitor at the power supply input for additional filtering.
</Note>

### Crystal Oscillator


## Programming Interface

### ICSP Header

Provide an In-Circuit Serial Programming (ICSP) header for code updates:

<ParamField path="PGD" type="Pin 40 (RB7)">
  Programming data line
</ParamField>

<ParamField path="PGC" type="Pin 39 (RB6)">
  Programming clock line
</ParamField>

<ParamField path="MCLR" type="Pin 1">
  Master clear (reset) line - requires pull-up resistor (10kΩ to VDD)
</ParamField>

<ParamField path="VDD, GND" type="Power pins">
  Supply programming voltage and ground
</ParamField>

### ICSP Connector Pinout

```
ICSP Header (5-pin):
  1. MCLR  (Reset)
  2. VDD   (+5V)
  3. GND   (Ground)
  4. PGD   (Data)
  5. PGC   (Clock)
```

<Note>
  RB6 and RB7 are shared between the keypad and ICSP. Ensure the keypad does not interfere during programming, or use jumpers to disconnect it temporarily.
</Note>

## PCB Design Considerations

<Note>
  When designing a PCB for this system, consider the following best practices:
</Note>

### Layout Guidelines

1. **Keep crystal close to MCU**: Minimize trace length between OSC1/OSC2 and crystal
2. **Ground plane**: Use a solid ground plane for noise reduction
3. **Bypass capacitors**: Place 0.1µF caps immediately adjacent to VDD pins
4. **LCD data bus**: Keep parallel data lines equal length if possible
5. **Keypad traces**: Row/column traces can be longer but avoid parallel routing

### Signal Integrity

<ParamField path="Clock traces" type="Short and direct">
  Crystal connections should be under 2 inches with ground guard traces
</ParamField>

<ParamField path="Digital ground" type="Single-point or plane">
  Connect all grounds at a single point or use a ground plane
</ParamField>

<ParamField path="LCD routing" type="Grouped and shielded">
  Route LCD control and data lines as a grouped bus with ground traces on either side
</ParamField>

## Troubleshooting

### Common Wiring Issues

<Accordion title="Keypad not responding">
  **Possible Causes:**
  - Pull-up resistors missing on row pins
  - Incorrect PORTB pin assignments
  - Row/column swapped
  
  **Solutions:**
  - Enable internal weak pull-ups or add external 10kΩ resistors
  - Verify pin definitions match physical wiring
  - Check continuity with multimeter
</Accordion>

<Accordion title="LCD displays garbage or nothing">
  **Possible Causes:**
  - Incorrect data bus connections
  - Control signals (RS, R/W, E, CS1, CS2) miswired
  - LCD contrast not adjusted
  - Power supply issues
  
  **Solutions:**
  - Verify DB0-DB7 connected to correct PORTD pins
  - Check control line definitions in hdm64gs12.c library
  - Adjust contrast potentiometer (typically VR1 on LCD module)
  - Verify 5V supply is stable and sufficient current available
</Accordion>

<Accordion title="Clock issues (program not running)">
  **Possible Causes:**
  - Crystal not oscillating
  - Wrong capacitor values
  - Fuse configuration incorrect
  
  **Solutions:**
  - Verify crystal with oscilloscope on OSC2
  - Use 15-33pF capacitors (check crystal datasheet)
  - Check FOSC fuse bits in device configuration
</Accordion>

## Testing Procedure

### Step-by-Step Verification

1. **Power Supply Test**
   - Measure voltage at VDD pins: should be 5.0V ±5%
   - Check for shorts between VDD and VSS

2. **Clock Verification**
   - Probe OSC2 with oscilloscope: should see ~8 MHz square wave
   - Measure frequency to verify clock accuracy

3. **LCD Test**
   - Program a simple test that fills screen with pattern
   - Verify all pixels can be addressed

4. **Keypad Test**
   - Program a test that displays key codes on LCD
   - Press each key and verify correct character displays

5. **Full System Test**
   - Load the complete Number Base Converter firmware
   - Test all three conversion modes
   - Verify calculations are accurate
